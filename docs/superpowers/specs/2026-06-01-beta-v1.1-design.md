# EventFinder Beta v1.1 — Design Spec

## Änderungen

### 1. Bug-Fix: set_proposals (500er bei Datumsänderung)
**Problem:** SQLAlchemy führt ORM-Deletes und Inserts in nicht garantierter Reihenfolge aus → UniqueViolationError auf `uq_event_date_proposal`.  
**Fix:** Ersetzt die Schleife `for p in existing: await db.delete(p)` durch ein direktes `DELETE WHERE event_id = ?` Statement (wird sofort geflusht bevor Inserts folgen).

### 2. Organisator-Benachrichtigung per E-Mail

**Trigger** (je eine Mail pro Aktion):
- Teilnehmer gibt Verfügbarkeit ab (`POST /events/{id}/availability`)
- Organisator ändert Terminvorschläge (`POST /events/{id}/proposals`)
- Neuer Teilnehmer eingeladen (`POST /events/{id}/invite`)

**Neuer Celery-Task:** `send_organizer_summary(event_id, action_text)`
- Holt aktuellen Stand aus DB (Proposals + Availabilities + Participants)
- Sendet Plain-Text-Mail an Organisator

**Mail-Format:**
```
Betreff: [Event-Titel] – Aktueller Stand

Aktion: [action_text]

TERMINÜBERSICHT
Sa, 13. Jun 2026   ✓✓ 2  ~ 1  ✗ 0  ? 2
...

TEILNEHMER (N)
✓ Name (Organisator) — 3/15 bewertet
○ Name              — 0/15 bewertet
```

**Implementierung:** Task wird async via `.delay()` nach dem DB-Commit aufgerufen (kein Warten auf Ergebnis). Gleiche Infrastruktur wie bestehende Mail-Tasks.

### 3. Persönlicher Einladungstext (optional)

**Scope:** Organisator kann beim Einladen eines Teilnehmers eine persönliche Nachricht eingeben.

**Frontend:** `InviteModal.vue` bekommt ein optionales `<textarea>` für die Nachricht.

**Backend:** 
- `POST /events/{id}/invite` akzeptiert jetzt `InviteRequest { email: str, message: str | None }`  
- `send_invitation_email` bekommt optionalen `message`-Parameter  
- Wenn gesetzt: Nachricht wird zwischen Betreff und Magic-Link in der Mail eingefügt

**Mail-Format mit Nachricht:**
```
Hallo,

[Organisator] hat dich zu "[Event]" eingeladen.

Persönliche Nachricht:
[message]

Klicke auf den folgenden Link um die Einladung anzunehmen:
[magic_link]
```
